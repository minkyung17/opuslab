API: Opus Case Data

Opus provides a RESTful web service to expose case data stored in our system. This is a read-only API that allows consumption of the data for integration into other systems for analytics, tracking, and reporting.

Working with the API

Requests listed below are prefixed with their necessary HTTP verb.

Making Valid Requests

Get valid authentication token
attach that token to every call you make from that time on

All requests should use the GET HTTP method. All API requests require HTTP Basic Authentication. Please see your systems administrator for assistance in determining the username and password for your site, or contact opushelp@conet.ucla.edu. It is strongly suggested that the following headers be set, or that all requests use a .json suffix to ensure that JSON responses are generated.

‘Accept’ HTTP Header

application/json

Recruit: Fetching Candidates for a given Job/Waiver or Exemption #

To list the entire set of candidates for a given string parameter Job Number / Waiver Number

GET https://opus.ucla.edu/opusService/getJobDetails?jobNumber=JPF00128
