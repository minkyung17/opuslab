UI Coding Standards

General

Naming conventions

Rule Example
Use lower case and hyphens
Good:
my-new-file.html

JavaScript

Syntax

Rule Example
Use === instead of ==. The same goes for !== instead of !=.
if (blah === "foo") {
Use double quotes for strings.
if (test === "success")

Whitespacing and Formatting

Rule Example
Use spaces before and after parentheses, comparators, etc.
Good:
if (blah === "foo") {
      foo("bar");
}
Bad:
if(blah==="foo"){
      foo("bar");
}
Same line braces
Good:
if (foo) {
      bar();
}
Bad:
if (foo)
{
      bar();
}
Always use braces
Good:
if (foo) {
      bar();
}
Bad:
if (foo)
      bar();
Don't mix tabs and spaces
Good:
if (foo) {
      bar();
      baz();
}
Bad:
if (foo) {
   bar();
      baz();
}
Line breaks - use the "+" at the end of the line, not at the beginning of the next
Good:
"foo()"+
"bar()"
Bad:
"foo()"
+"bar()"
Consistent indentation - line up opening and closing tags, curly braces, etc.
Good:
if (foo) {
      bar();
}
Bad:
if (foo) {
      bar();
      }

Variables

Rule Example
Use camel case
var xPosition = 1;

CSS

Spacing

Rule Example
Use proper spacing
Good:
.selector {
      display: none;
      background: #ff0000;
      color: #000000;
}
Bad:
.selector {display:none;background:#ff0000;color:#000000;}
